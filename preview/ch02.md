# 第2章：四大入口全面拆解 — 網頁 / 桌面 / 手機 / CLI

---

## ① 30 秒摘要

**這一章講什麼？**
Claude 有四個入口：網頁版（claude.ai）、桌面版（Desktop App）、手機版（iOS/Android）、命令列版（Claude Code CLI）。每個入口有不同的功能和限制。選錯入口，你會錯過 Claude 50% 的能力。

**為什麼重要？**
大部分人只知道 claude.ai 網頁版。但桌面版有 Cowork 代理模式和 Computer Use；手機版有語音和相機；CLI 有 Git 深度整合、Hooks 自動化、Plugins。選對入口 = 效率翻倍。

**讀完能做什麼？**
根據你的任務類型，3 秒選出最適合的入口，並知道每個入口的獨佔功能。

### 2.2 網頁版（claude.ai）                                       【Web】

**一句話定位：** 最易上手、功能最全面的通用入口。

#### 核心功能

**Projects（專案）**
- 把相關的對話、文件、設定組織在一起
- 上傳 Project Knowledge（30MB/檔案，20 檔/對話）
- 寫 Custom Instructions（等同 CLI 的 CLAUDE.md）
- 團隊版可共享 Projects

**Artifacts（成品面板）**
- Claude 的輸出如果是程式碼、HTML、圖表、文件，會顯示在獨立面板
- 8 種類型：程式碼、HTML/CSS/JS、React、SVG、Mermaid 流程圖、Markdown、可下載檔案
- 可以直接在面板裡編輯和預覽

**Styles（風格）**
- 4 種預設：Normal / Concise / Formal / Explanatory
- 可以自訂風格（用文字描述或上傳寫作範本）

**MCP 連接器（38+ 個）**
- 一鍵連接 Google Drive、Slack、Notion、GitHub 等
- 不需要技術設定

**Web Search（網路搜尋）**
- Claude 可以即時搜尋網路，附上引用來源
- 自動判斷何時需要搜尋

#### 最適合場景

| 場景 | 為什麼用網頁版 |
|------|---------------|
| 寫文章、報告 | Artifacts 面板即時預覽 |
| 分析 PDF 文件 | 直接上傳，支援 100 頁以內 |
| 團隊協作 | Projects 共享功能 |
| 快速問答 | 打開瀏覽器就能用 |
| 連接雲端服務 | 38+ 內建 MCP 連接器 |

---

### 2.3 桌面版（Desktop App）                                     【Desktop】

**一句話定位：** 網頁版的所有功能 + 本地能力 + AI 代理。

#### 桌面版獨佔功能

**Cowork（代理模式）**
- 白話：你丟一個資料夾給 Claude，它自己處理
- 拖放檔案或資料夾 → Claude 自主分析、修改、生成
- 適合批次處理和自動化任務

**Computer Use（電腦操控）**
- Claude 可以操控你的桌面 GUI——點擊、打字、截圖
- 適合操作沒有 API 的舊軟體
- 需要 Pro 或 Max 方案

**語音模式**
- 5 種語音人格
- 按住說話或連續聆聽模式
- 適合通勤、做家事時使用

**本地 MCP 伺服器**
- 透過 Desktop Extensions 一鍵安裝
- 或手動設定 `claude_desktop_config.json`
- 可以存取本地檔案系統、資料庫（→ Ch20）

**系統快捷鍵**
- macOS：雙擊 `Option` 叫出 Quick Entry
- Windows：`Alt+Space` 叫出 Quick Entry
- 任何時候都能快速問 Claude

**排程任務**
- 設定定時任務，Claude 自動執行
- 適合每日報告、定期資料整理

#### 最適合場景

| 場景 | 為什麼用桌面版 |
|------|---------------|
| 操作本地檔案 | 直接存取，不用上傳 |
| 需要 AI 代理 | Cowork 模式自主工作 |
| 操控桌面 GUI | Computer Use |
| 語音互動 | 5 種語音人格 |
| 連接本地資料庫 | 本地 MCP 伺服器 |

### 2.4 手機版（iOS / Android）                                   【Mobile】

**一句話定位：** 碎片時間的 Claude 入口。

#### 手機版功能

- **所有網頁版功能**（Projects、Artifacts 檢視、Styles、MCP 連接器）
- **語音輸入** — 說話比打字快 3 倍
- **相機** — 拍照讓 Claude 分析（文件、白板、實物）
- **Generative UI** — 互動式視覺化（2026 年 3 月上線）

#### 手機版沒有的功能

- ❌ Cowork 代理模式
- ❌ 本地 MCP 伺服器
- ❌ Computer Use
- ❌ Desktop Extensions
- ❌ Skills

#### 最適合場景

| 場景 | 為什麼用手機版 |
|------|---------------|
| 通勤時語音問答 | 語音輸入最方便 |
| 拍照分析 | 相機 + Vision |
| 碎片時間處理簡單任務 | 隨手可用 |
| 在外面監控 AI 任務 | 查看 Claude 的回覆 |

### 2.5 CLI（Claude Code）                                        【CLI】

**一句話定位：** 開發者的終極武器。

#### CLI 獨佔功能

| 功能 | 說明 | 其他入口 |
|------|------|---------|
| **Git 深度整合** | 自動讀取 git diff、commit、PR | ❌ |
| **多檔案並行編輯** | 一次修改 10 個檔案 | 逐一 |
| **Hooks 自動化** | 在工具呼叫前/後執行自訂腳本 | ❌ |
| **Plugins** | 32 官方 + 17 外部 + 400+ 社群 | ❌ |
| **Skills** | 按需載入的專業知識包 | 有限 |
| **CLAUDE.md** | 專案級的 AI 設定檔 | Custom Instructions |
| **費用追蹤** | `/cost` 即時看花了多少錢 | ❌ |
| **Headless 模式** | `-p` 參數用在腳本和 CI/CD 中 | ❌ |
| **Worktree 隔離** | `claude -w` 在 Git worktree 中工作 | ❌ |
| **Agent / 子代理** | 啟動多個 Agent 並行工作 | ❌ |

#### 最適合場景

| 場景 | 為什麼用 CLI |
|------|-------------|
| 寫程式 / 改程式 | Git 整合 + 多檔案編輯 |
| 程式碼審查 | `/security-review` + code-review plugin |
| 自動化工作流 | Hooks + Headless + CI/CD |
| 大型專案開發 | CLAUDE.md + Skills + Worktree |
| 團隊開發 | 設定檔 Git 共享 |

> TIP：如果你是開發者，Claude Code CLI 是使用 Claude 最強大的方式。它有其他三個入口都沒有的獨佔功能。本書 PART 2-6 的大部分內容都圍繞 CLI 展開。

### 2.6 入口選擇決策流程

```
你要做什麼？
│
├─ 寫程式 / 改程式 / 程式碼審查
│   └── → CLI（Claude Code）
│       Git 整合 + 多檔案 + Plugins + 費用追蹤
│
├─ 寫文章 / 分析報告 / 做簡報
│   └── → 網頁版（Artifacts 面板）
│       即時預覽 + Projects 組織 + Styles
│
├─ 操作電腦 GUI / AI 代理任務
│   └── → 桌面版（Computer Use + Cowork）
│       本地存取 + 自主代理
│
├─ 通勤 / 碎片時間 / 拍照分析
│   └── → 手機版（語音 + 相機）
│       隨時隨地
│
├─ 團隊協作
│   └── → 網頁版（Projects 共享）
│       或 CLI（.claude/ 設定檔 Git 共享）
│
└─ 自動化 / CI-CD / 批次處理
    └── → CLI（Headless 模式 + Hooks）
        腳本化 + 排程
```

### 2.7 入口間的串接

#### 共用的部分

| 功能 | 跨入口共用？ | 說明 |
|------|------------|------|
| 帳號和方案 | ✅ | 同一帳號登入所有入口 |
| 記憶 | ✅ | Memory 跨平台同步 |
| MCP 遠端連接器 | ✅ | 雲端連接器跨平台可用 |
| API Key | ✅ | 同一把 Key 用在 CLI 和 API |

#### 不共用的部分

| 功能 | 說明 |
|------|------|
| Projects ≠ CLAUDE.md | 網頁版的 Custom Instructions 和 CLI 的 CLAUDE.md 是獨立系統 |
| MCP 本地伺服器 | 桌面版和 CLI 各自設定，不互通 |
| Plugins | CLI 獨佔 |
| 對話歷史 | 網頁/桌面/手機共用，CLI 獨立 |

> WARNING：網頁版的 Projects Custom Instructions 和 Claude Code 的 CLAUDE.md 是**完全獨立的兩個系統**。在網頁版設定的指令不會影響 CLI，反之亦然。如果你兩邊都用，需要分別設定。

## ⑥ 常見陷阱

> WARNING：選擇入口時最常犯的 3 個錯誤

### 陷阱一：只用網頁版

**問題：** 90% 的用戶只知道 claude.ai。錯過了桌面版的 Cowork、CLI 的 Git 整合等強大功能。
**解法：** 至少試用一次桌面版和 CLI。花 10 分鐘安裝 Claude Code（→ Ch6），你會發現一個全新的世界。

### 陷阱二：開發者不用 CLI

**問題：** 開發者在網頁版裡複製貼上程式碼，來回 10 次。
**解法：** 安裝 Claude Code CLI。它能直接讀你的程式碼、改你的檔案、跑你的測試——不需要複製貼上。

### 陷阱三：以為各入口功能一樣

**問題：** 以為在手機上也能用 Computer Use 或 Cowork。
**解法：** 看 2.1 的對比總表。每個入口有不同的獨佔功能。

## ⑦ 要點回顧

### 3 個關鍵要點

1. **四大入口各有專長** — 網頁版最通用、桌面版有 AI 代理、手機版有語音相機、CLI 是開發者終極武器
2. **CLI 獨佔功能最多** — Git 整合、Hooks、Plugins、Skills、Headless、費用追蹤——這些其他入口都沒有
3. **入口間不完全互通** — 記憶和帳號共用，但 Projects/CLAUDE.md、本地 MCP、對話歷史各自獨立
