---

## ① 30 秒摘要

**這一章講什麼？**
Claude 有 6 種付費方案（Free / Pro / Max 5x / Max 20x / Team / Enterprise）加上獨立計費的 API。本章拆解每個方案的費用、用量、解鎖功能，並教你三個省錢武器：Prompt Caching（省 90%）、Batch API（省 50%）、兩者疊加（省 95%）。

**為什麼重要？**
選錯方案 = 多花很多冤枉錢。很多人把 Max 理解成「無限量」，但官方其實是按 Pro 的 5x / 20x 使用級距區分；也很多人以為 Prompt Caching 會自動幫你省錢，卻沒注意 API 端要正確開啟 cache 控制。

**讀完能做什麼？**
根據你的使用量選對方案，用三個省錢武器把 API 帳單壓到最低。

### 3.3 API 定價精算

> API 和訂閱方案是**獨立計費**的。訂閱方案買的是 claude.ai / Desktop / Mobile 的額度；API 費用另外算。Claude Code CLI 消耗的是你的訂閱額度（如果是 Pro/Max）或 API 額度。

#### 三大模型定價

| 模型 | 輸入 / MTok | 輸出 / MTok | Caching 寫入 | Caching 讀取 |
|------|------------|------------|-------------|-------------|
| Opus 4.6 | $5 | $25 | $6.25 | $0.50 |
| Sonnet 4.6 | $3 | $15 | $3.75 | $0.30 |
| Haiku 4.5 | $1 | $5 | $1.25 | $0.10 |

#### 白話翻譯

| 操作 | Opus | Sonnet | Haiku |
|------|------|--------|-------|
| 讀 100 頁 PDF（~150K tokens） | $0.75 | $0.45 | $0.15 |
| 寫 10 頁報告（~15K tokens） | $0.38 | $0.23 | $0.08 |
| 一次完整對話（問+答） | $0.50-2.00 | $0.10-0.50 | $0.02-0.10 |
| 審查一個 PR | $0.30-1.00 | $0.10-0.30 | — |

> TIP：**輸出比輸入貴 5 倍**。讓 Claude 寫長文很貴（輸出多），讓 Claude 分析你的長文件很便宜（輸入多）。如果你想省錢，可以要求 Claude「用簡潔格式回覆」——這直接減少輸出 token。

### 3.4 省錢武器一：Prompt Caching                                 COST SAVE

#### 白話解釋

你每次跟 Claude 對話時，都要把 System Prompt 和之前的對話歷史「重新傳送」一次。這些重複的內容每次都要付費。

Prompt Caching 的作用：**把這些重複的內容存起來，下次只付 10% 的費用。**

```
沒有 Caching：
對話 1：送 System Prompt（6,000 tokens）→ 付 100%
對話 2：送 System Prompt（6,000 tokens）→ 付 100%
對話 3：送 System Prompt（6,000 tokens）→ 付 100%
→ 三次共付 18,000 tokens 的輸入費

有 Caching：
對話 1：送 System Prompt（6,000 tokens）→ 付 100%（寫入快取）
對話 2：讀取快取（6,000 tokens）→ 付 10%
對話 3：讀取快取（6,000 tokens）→ 付 10%
→ 三次共付 6,000 + 600 + 600 = 7,200 tokens → 省 60%
```

#### 省錢幅度

| 場景 | 重複比例 | 節省 |
|------|---------|------|
| 長 System Prompt + 多輪對話 | 高 | 60-90% |
| 批次處理（同一指令 + 不同輸入） | 很高 | 80-90% |
| 分析多個檔案（同一分析框架） | 高 | 70-85% |
| 每次都不同的一次性問答 | 低 | 5-15% |

> TIP：API 端的 Prompt Caching **不是完全零設定**。你要在請求裡加 `cache_control`（推薦）或明確標註 cache breakpoint，才會按快取價格計費。Claude Code / Claude app 內部會自行管理，但自己串 API 時要顯式開啟。

### 3.5 省錢武器二：Batch API                                     COST SAVE

#### 白話解釋

Batch API 讓你把一堆任務打包送出，Anthropic 在 24 小時內處理完——作為交換，你付半價。

```
即時 API：
送出 → 等 3 秒 → 收到結果
費用：100%

Batch API：
送出 100 個任務 → 等幾小時 → 收到所有結果
費用：50%
```

#### 適合場景

| 場景 | 即時 API | Batch API |
|------|---------|-----------|
| 即時對話 | ✅ 必須用 | ❌ 太慢 |
| PR 審查（需要馬上看到結果） | ✅ | ❌ |
| 翻譯 1,000 篇文章 | ❌ 太貴 | ✅ 半價 |
| 分類 10,000 封郵件 | ❌ 太貴 | ✅ 半價 |
| 批次生成測試案例 | 看情況 | ✅ 半價 |
| 每日定期報告 | 看情況 | ✅ 半價 |

> TIP：**不需要即時回應的任務，永遠用 Batch API。** 同樣的結果，一律半價。

---

### 3.6 省錢武器三：Caching + Batch 疊加                           COST SAVE

**殺手級組合：** Prompt Caching + Batch API 可以疊加使用。

```
原價：$1.00
└─ 加 Prompt Caching：$0.10（省 90%）
   └─ 再加 Batch API：$0.05（再省 50%）

總計：從 $1.00 → $0.05 → 省 95%
```

#### 真實範例

| 任務 | 原價 | + Caching | + Batch | 最終 | 節省 |
|------|------|----------|---------|------|------|
| 1,000 篇文章摘要 | $50 | $10 | $5 | $5 | 90% |
| 10,000 封郵件分類 | $100 | $15 | $7.50 | $7.50 | 92.5% |
| 500 份合約審查 | $250 | $50 | $25 | $25 | 90% |

> TIP：這個組合適合「同一個指令 + 大量不同輸入」的場景。System Prompt 和指令模板被 Caching，每個輸入的不同部分正常計費，再用 Batch 打五折。

## ⑥ 常見陷阱

> WARNING：選方案和控成本時最常犯的 3 個錯誤

### 陷阱一：不知道 Pro 和 API 是分開計費的

**問題：** 以為付了 $20 Pro 就能無限呼叫 API。
**解法：** Pro/Max 是 claude.ai / Desktop / Mobile / Claude Code 的額度。API 是按 token 另外計費。兩者獨立。

### 陷阱二：Free 方案碰到限額就放棄

**問題：** Free 方案幾個小時就碰到限額，覺得「Claude 不好用」。
**解法：** Free 的限額極低。升級到 Pro $20 後用量提升 5 倍，體驗完全不同。試用 Claude 的正確方式是至少用一個月 Pro。

### 陷阱三：不知道 Batch API 存在

**問題：** 每天手動送 100 個 API 請求，每個都是即時的，帳單是本來的 2 倍。
**解法：** 不需要即時回應的任務一律用 Batch API——同樣品質，一律半價。
